Servicios

Los servicios en Angular nos permiten manejar la informacion que sera presentada un la aplicación web. Los servicios realizan consultas HTTP a WebServices Locales (Corporativos, propios de la empresa) o a APIs de algun servidor externo a la nube que puede contener datos publicos como privados.

Para crear un servicio usando Angular CLI, usamos el siguente comando:

Servicio mediante CLI de Angular
>>> ng g s _services/DataServices

Peticiones HTTP Lo basico de lo basico

Anteriormente se menciono que los servicios pueden hacer uso de recursos externos para consulta la informacion. Estos recursos externos pueden ser APIs publicas o un webservice que exponga un API Rest para consumir información.

Angular provee al desarrollador un mecanismo sencillo de realizar peticiones HTTP. Para esto Angular hace uso del modulo HttpClientModule, del cliente HttpClient y de los observables de la libreria RxJs. Estos componentes junto con otro llamado INTERCEPTOR proveen al desarrollador lo suficiente para comunicarse con el backend y enviar o consultar información.

El modulo HttpClientModule debe ser importado solo en el modulo principal app.module.ts, esto no es necesario para la utilidad HttpClient y la libreria RxJS.

Pipes

Los pipes nos permiten transformar los datos de forma visual. Para indicar a angular que se usara un Pipe se debe usar el simbolo de tuberia ('|'). Por lo general se usa por el lado de HTML.

Por ejemplo, si queremos transformar un texto a mayusculas, hariamos lo siguiente:

TEXTO A MAYUSCULAS
>>> texto = 'Hola Mundo'
>>> {{ texto | uppercase }}
>>> out: HOLA MUNDO

El pipe UPPERCASE es uno de los muchos que existen, a continuacion una lista de algunos pipes y una breve descripción:

PIPE Descripcion
slice Permite cortar textos, es parecido a la funcion substring en otros lenguajes, tambien trabaja sobre arreglos. Este pipe al igual que otros recibe parametros indicados con dos puntos despues del nombre del pipe.
decimal Este pipe nos permite representar cantidades numericas, indicando la cantidad de enteros y decimales.
uppercase Permite llevar un texto a mayusculas
lowercase Permite llevar un texto a minusculas
percent Nos permite transformas cantidades numericas a representaciones porcentuales
json Nos permite formatear un objeto de TS a tipo JSON.
async Desencadena un valor de una primitiva asincronica. No recibe argumentos. Este pipe debe tomar una expresion de tipo asincronica que puede ser un observable o una promesa.
date Permite trabajar con fechas.

Storage (Local y Sessin Storage)

Ambos son metodos para el almacenamiento local de información. La diferencia entre Local y Session Storage es que Local Storage mantiene la información almacenada hasta que el navegador del usuario sea cerrada o la aplicación limpie la data. Al contrario Session Storage mantiene la información hasta cerrar la pestaña a la se limpie la data del navegador.

El uso de Local y Session Storage es identico, en este punto mostraremos el uso basico de LocalStorage. LocalStorage hace uso del objeto localStorage el cual tiene los siguientes metodos para manejar la información:

METODO DESCRIPCION
localStorage.setItem(key, value) Permite almacenar informacion en local storage, key es el nombre de la variable o dato a almacenar y value, logicamente, es el valor a asignar en dicha variable.
localStorage.getItem(key) A diferencia del anterior, getItem(key) obtiene el valor de una variable almacenada en el localStorage.
localStorage.removeItem(key) Este metodo nos permite remover una variable almacenada en el localStorage, para esto es necesario pasarle el nombre de la variable o key como parametro.
localStorage.clear() Permite limpiar toda la informacion almacenada en el localStorage.
